# 프레스킷 관리자 사용 가이드

Hillstone Partners 프레스킷 관리자 페이지 사용 방법입니다.

## 🚀 빠른 시작

### 1단계: 관리자 로그인

1. `newsroom-admin.html` 페이지 열기
2. 비밀번호 입력: `admin123`
3. "로그인" 버튼 클릭

### 2단계: GitHub 설정 (최초 1회만)

1. 상단 **"GitHub 설정"** 버튼 클릭
2. [GitHub Token 생성](GITHUB_TOKEN_SETUP.md) 가이드 참고
3. 설정 정보 입력:
   - **GitHub Token**: 생성한 Personal Access Token
   - **Repository Owner**: `harhie`
   - **Repository Name**: `hillstone-presskit`
   - **Branch**: `main`
4. **"설정 저장"** 클릭

### 3단계: 기사 관리

#### 새 기사 추가
1. **"+ 새 항목 추가"** 버튼 클릭
2. 기사 정보 입력:
   - 제목 (필수)
   - 카테고리 (필수): 기사/해외기사/공지/영상
   - 날짜 (필수)
   - 출처 (필수)
   - 요약 (선택)
   - 링크 URL (선택)
   - 이미지 URL (선택, 공지는 여러 장 가능)
3. **"저장"** 클릭

#### 기사 수정
1. 기사 목록에서 수정할 기사의 **✏️ 수정** 버튼 클릭
2. 내용 수정
3. **"저장"** 클릭

#### 기사 삭제
1. 기사 목록에서 삭제할 기사의 **🗑️ 삭제** 버튼 클릭
2. 확인 창에서 **"확인"** 클릭

### 4단계: GitHub 동기화

⚠️ **중요**: 기사 추가/수정/삭제 후 반드시 동기화해야 공개 페이지에 반영됩니다!

1. 상단 **"GitHub에 동기화"** 버튼 클릭
2. 동기화 진행 (5-10초 소요)
3. 완료 메시지 확인
4. 1-2분 후 공개 페이지에서 변경사항 확인

## 📊 대시보드

### 통계 카드
- **전체 항목**: 모든 기사 총 개수
- **기사**: 국내 기사 개수
- **공지**: 공지사항 개수
- **영상**: 영상 개수

### 기사 목록
- 날짜 순으로 정렬 (최신순)
- 제목, 카테고리, 날짜, 출처 표시
- 수정/삭제 버튼

## 🖼️ 이미지 관리

### 일반 기사/해외기사/영상
- 이미지 1장만 추가 가능
- 이미지 URL 직접 입력

### 공지 카테고리
- 여러 장의 이미지 추가 가능
- **"+ 이미지 URL 추가"** 버튼으로 추가
- **✖️** 버튼으로 삭제 (최소 1장 유지)

### 이미지 호스팅 추천
- [Imgur](https://imgur.com) - 무료, 간단
- [GitHub](https://github.com) - 저장소에 업로드 후 Raw URL 사용
- [Cloudinary](https://cloudinary.com) - 전문적인 이미지 관리

## ⚙️ 작동 원리

### LocalStorage (임시 저장)
- 어드민에서 추가/수정/삭제한 내용은 **브라우저 LocalStorage**에 저장
- 같은 브라우저에서만 보임
- 다른 사용자나 기기에서는 보이지 않음

### GitHub 동기화 (영구 저장)
- **"GitHub에 동기화"** 버튼 클릭 시:
  1. LocalStorage 데이터 읽기
  2. `js/main.js` 파일의 `getSampleData()` 함수 업데이트
  3. GitHub 저장소에 커밋
  4. GitHub Pages 자동 배포
- 동기화 후 **모든 사용자**에게 변경사항 반영

## 🔒 보안

### GitHub Token 관리
- ✅ LocalStorage에 안전하게 저장
- ✅ 비밀번호 타입 입력란 (화면에 숨김)
- ❌ 절대 코드에 하드코딩하지 마세요
- ❌ 다른 사람과 공유하지 마세요

### Token이 노출된 경우
1. GitHub Settings → Developer settings
2. Personal access tokens
3. 해당 토큰 **Delete**
4. 새 토큰 재생성
5. 관리자 페이지에서 재설정

## 🐛 문제 해결

### "GitHub 설정이 필요합니다" 오류
- **원인**: GitHub Token이 설정되지 않음
- **해결**: "GitHub 설정" 버튼 클릭 → Token 입력

### "GitHub API Error: 401" 오류
- **원인**: Token이 잘못되었거나 만료됨
- **해결**: 새 Token 생성 → 재설정

### "GitHub API Error: 404" 오류
- **원인**: Repository 정보가 잘못됨
- **해결**: Owner/Repo/Branch 정보 확인

### 동기화 후 변경사항이 안 보임
- **원인**: GitHub Pages 배포 지연
- **해결**: 1-2분 기다린 후 새로고침
- **또는**: 브라우저 캐시 삭제 (Ctrl+Shift+R)

### "getSampleData 함수를 찾을 수 없습니다" 오류
- **원인**: `js/main.js` 파일 구조 변경됨
- **해결**: 개발자에게 문의

## 📝 워크플로우 예시

### 시나리오: 새 기사 10개 추가

1. **관리자 로그인**
2. **기사 추가** (10번 반복)
   - "+ 새 항목 추가" 클릭
   - 정보 입력
   - 저장
3. **미리보기**: "뉴스룸 보기" 버튼으로 확인 (LocalStorage)
4. **GitHub 동기화**: 모든 작업 완료 후 1번만 실행
5. **확인**: 1-2분 후 공개 페이지에서 확인

### 💡 Tip
- 여러 기사를 추가/수정한 후 **한 번에 동기화**하세요
- 매번 동기화하면 GitHub API 제한에 걸릴 수 있습니다
- 하루 최대 동기화 권장 횟수: 10회

## 🎯 모범 사례

### 기사 작성
- ✅ 제목은 명확하고 간결하게
- ✅ 요약은 2-3줄로 핵심만
- ✅ 출처는 정식 매체명 사용
- ✅ 링크는 원문 URL 직접 사용
- ✅ 날짜는 기사 발행일 기준

### 이미지 사용
- ✅ 고해상도 이미지 (최소 1200px 너비)
- ✅ 웹 최적화 포맷 (JPEG, PNG, WebP)
- ✅ 안정적인 호스팅 서비스 사용
- ❌ 저작권 있는 이미지 무단 사용 금지

### 카테고리 선택
- **기사**: 국내 언론사 기사
- **해외기사**: 해외 언론사 기사
- **공지**: 행사, MOU, 수상 등 (사진 여러 장)
- **영상**: 영상 콘텐츠 링크
